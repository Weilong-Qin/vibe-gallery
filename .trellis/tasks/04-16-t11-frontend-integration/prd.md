# T11: Frontend Integration (App.tsx + section ordering)

## Goal
Wire the React frontend to consume `gallery.json` and render the full portfolio page using all components from T06-T09. Replace the demo placeholder App.tsx with the real data-driven app.

## Requirements

### `src/app/App.tsx` — Full implementation
```tsx
import React from 'react'
import galleryData from './data/gallery.json'
import type { GalleryData } from '../../types/index.js'
import { Layout } from './components/Layout.js'
import { Profile } from './components/resume/index.js'
import { Skills } from './components/resume/index.js'
import { Experience } from './components/resume/index.js'
import { Education } from './components/resume/index.js'
import { ProjectGrid } from './components/projects/index.js'

const data = galleryData as unknown as GalleryData

export default function App() {
  const { profile, resume, projects, theme, accent, layout } = data

  const resumeSections = resume.sections ?? ['skills', 'experience', 'education', 'projects']

  const sectionComponents: Record<string, React.ReactNode> = {
    skills: resume.skills?.length ? <Skills key="skills" skills={resume.skills} /> : null,
    experience: resume.experience?.length ? <Experience key="experience" experience={resume.experience} /> : null,
    education: resume.education?.length ? <Education key="education" education={resume.education} /> : null,
    projects: <ProjectGrid key="projects" projects={projects} />,
  }

  const isSidebar = layout.page === 'sidebar'

  return (
    <Layout layout={layout} theme={theme} accent={accent}>
      {isSidebar ? (
        <div className="page-root">
          <aside style={{ position: 'sticky', top: 'var(--space-xl)', alignSelf: 'start' }}>
            <Profile profile={profile} />
            {resumeSections
              .filter(s => s !== 'projects')
              .map(s => sectionComponents[s])
              .filter(Boolean)}
          </aside>
          <main>
            {sectionComponents['projects']}
          </main>
        </div>
      ) : layout.page === 'hero' ? (
        <>
          <div className="hero-section">
            <Profile profile={profile} />
          </div>
          <div className="content-section">
            <div className="page-root">
              {resumeSections
                .filter(s => s !== 'projects')
                .map(s => sectionComponents[s])
                .filter(Boolean)}
              {sectionComponents['projects']}
            </div>
          </div>
        </>
      ) : (
        <div className="page-root">
          <Profile profile={profile} />
          {resumeSections.map(s => sectionComponents[s]).filter(Boolean)}
        </div>
      )}
    </Layout>
  )
}
```

### `vite.config.ts` update
Add JSON import support (Vite supports JSON by default, but ensure `resolveJsonModule` is also in tsconfig):

In `tsconfig.json`, add `"resolveJsonModule": true` to compilerOptions.

### `src/app/data/gallery.json`
This file is generated by `npm run build:data`. For development, ensure a valid demo JSON exists (it's already created by T10).

### Section spacing
Add global section spacing to `src/app/themes/base.css`:
```css
section + section {
  margin-top: var(--space-xl);
}

aside > section + section,
.page-root > section + section {
  margin-top: var(--space-xl);
}
```

- [ ] Sidebar layout renders Profile+resume in left column, projects in right
- [ ] Hero layout renders Profile full-width then content below

## Technical Notes
- Import `gallery.json` directly — Vite handles JSON imports natively
- Use `as unknown as GalleryData` cast since JSON import type is generic
- Section ordering follows `resume.sections` array from config (user-controlled)
- If `gallery.json` doesn't exist, `npm run dev` will fail — that's acceptable (user must run `build:data` first or use the demo JSON from T10)
